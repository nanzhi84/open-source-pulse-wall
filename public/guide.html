GitHub 分级教学目录

这套教程面向第一次参与开源协作的学生。它不要求学生一开始就理解所有命令，而是先建立协作地图，再分别练习网页端、VS Code、命令行和 Pull Request 修错。左侧目录保持固定，学生可以像读书一样按章节推进。

学习路径

1. GitHub 协作地图 先把 Repository、Issue、Fork、Branch、Commit、Pull Request 放到同一张图里，避免后续操作变成死记按钮。 2. GitHub 网页端贡献 只用浏览器完成一次 Profile 贡献，让学生先获得完整闭环，再补本地工具。 3. VS Code 全流程 用文件树、内置终端、JSON 校验和 Source Control 面板减少路径错误、格式错误和漏提交。 4. 命令行路径 把 clone、branch、validate、commit、push 对应到本地文件、Git 历史和 GitHub 页面变化。 5. PR Review 与修错 讲清楚检查失败、review 评论、追加提交和同步 fork，不让学生把 PR 当成一次性提交。 6. Profile 字段实验 逐个解释 `data/profiles/你的用户名.json` 的字段、格式、校验和常见错误。

建议课时

章节 建议视频 课堂目标
1. 协作地图3 分钟让学生知道自己为什么要 Fork、开分支和发 PR。
2. 网页端贡献6 分钟不依赖本地环境，完成第一次贡献闭环。
3. VS Code 全流程8 分钟用可视化工具减少路径、JSON 和提交范围错误。
4. 命令行路径8 分钟把每条 Git 命令和可观察结果对应起来。
5. PR Review 与修错6 分钟学会读检查结果、回应 review、在原 PR 上继续修。
6. 字段实验6 分钟把 Profile JSON 写对，能自己定位格式错误。

教师使用建议

  • 第一节课只要求完成第 1 到第 3 章。学生能在 VS Code 里通过校验，比背完所有 Git 命令更重要。
  • 第 4 章适合放在第二次练习前。学生已经见过 Fork 和 PR 后，再解释命令行会更容易。
  • 第 5 章不要提前讲太多，最好等学生第一次看到失败检查或 review 评论后再讲。
  • 第 6 章可以作为随堂检查表。只要 JSON 出错，就回到这一章逐项排查。
