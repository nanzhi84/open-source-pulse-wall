6. Profile 字段实验

这个实验专门处理 `data/profiles/你的用户名.json`。学生只改自己的 JSON 文件，先保证格式正确，再考虑个人表达。所有字段都要按项目规则填写，否则展示墙可能无法读取。

推荐模板

{
  "name": "你的名字",
  "github": "your-github-id",
  "role": "First-time contributor",
  "motto": "今天完成我的第一个开源 PR",
  "stack": ["Git", "Open Source"],
  "city": "Beijing",
  "style": "nature",
  "avatar": "avatars/avatar-01.png",
  "homepage": "",
  "cheers": []
}

复制模板后，先只改文本内容，不要调整字段顺序，也不要删除字段。新手阶段保持结构稳定，排错会简单很多。

字段说明

`name`

展示在卡片上的名字，可以是真名、昵称或课堂称呼。不要留空。

`github`

GitHub 用户名。建议和文件名保持一致，例如文件名是 `octocat.json`，这里就写 `octocat`。

`role`

身份或标签，例如 `First-time contributor`、`学生`、`Frontend learner`。它应该短一些，方便卡片展示。

`motto`

一句话介绍自己。可以写学习目标、参与原因或一句简短宣言，不要写太长。

`stack`

技术标签数组。每个标签都要放在英文双引号里，标签之间用英文逗号分隔。

`city`

城市或课堂位置，可以写中文。这个字段用于展示个人来源，不需要写详细地址。

`style`

卡片风格，只能使用项目支持的值：`minimal`、`nature`、`sketch`、`notebook`、`ink`、`sage`。

`avatar`

头像路径。可以使用内置头像，例如 `avatars/avatar-01.png`，也可以使用安全的 https 图片链接。

`homepage`

个人主页链接。没有主页就写空字符串 `""`，不要随便填不完整的网址。

`cheers`

寄语列表。第一次添加 Profile 时保持空数组 `[]`，这个字段留给其他同学之后给你写寄语。

JSON 格式规则

  • 字符串必须使用英文双引号，不能用中文引号，也不要用单引号。
  • 字段名和字段值之间用英文冒号。
  • 字段之间用英文逗号分隔，但最后一个字段后面不能有逗号。
  • 数组用方括号，例如 `["Git", "Open Source"]`。
  • 空字符串写成 `""`，空数组写成 `[]`。
  • 保存后运行 `npm run validate`，不要只相信肉眼检查。

常见错误和修法

最后一项多了逗号

把最后一个字段后面的逗号删掉。JSON 不允许尾随逗号。

文件名不是 GitHub 用户名

把文件重命名为自己的 GitHub 用户名，例如 `octocat.json`。

改到了 `_template.json`

恢复模板文件，再复制出自己的新文件。模板不属于任何一个学生。

`stack` 写成普通文本

`stack` 应该是数组，不是 `"Git, Open Source"` 这种字符串。

`style` 写了不存在的值

回到支持列表里选择一个，例如 `nature` 或 `notebook`。

自己填写了 `cheers`

第一次提交时保持 `cheers: []`，寄语由其他同学后续添加。

提交前自检

npm run validate

如果校验失败，先看报错里出现的文件名，再看字段名。不要一次性改很多地方；每次只修一个明确错误，保存后重新运行校验。
