    课堂流程

    1

    Fork 与 Clone

    Fork 本仓库到你的 GitHub，并 Clone 到本地。

    2

    Branch 与 Commit

    创建个人分支，添加你的信息并提交 Commit。

    3

    Push 与 Pull Request

    推送到远程仓库，并创建 Pull Request 到 main 分支。

    4

    Review 与 Merge

    通过代码审查，合并后你的卡片展示在贡献墙。

    冲突处理 5 步法

    PR 提示 conflicts must be resolved 时按下面顺序操作。默认 merge，rebase 进阶。

    1. 1

      Sync

      拉老师仓库

    2. 2

      Edit

      删冲突标记

    3. 3

      Check

      npm run validate

    4. 4

      Save

      git add + commit

    5. 5

      Push

      推到自己分支

    完整命令（默认 merge 流程） 
    # 1. Sync 同步老师仓库
git fetch upstream
git checkout add-your-profile
git merge upstream/main

# 2. Edit 打开冲突文件，删掉 <<<<<<< / ======= / >>>>>>> 三行
#    把两边内容合并成合法 JSON

# 3. Check 校验
npm run validate

# 4. Save 提交
git add <冲突文件>
git commit

# 5. Push 推送
git push origin add-your-profile

    进阶 rebase：第 1 步把 merge 换成 rebase，遇到冲突修完后 git rebase --continue，第 5 步用 git push --force-with-lease（不要 -f）。

    给同学加油 (Cheers)

    在对方的 cheers 数组里追加 { from, message }。多人同时加油会触发冲突，正好练习上面五步。

    提一个好 Issue

    四种模板可选。标题「动词+现象」，复现步骤一步步写，长日志用 ``` 包起来。

    请 Reviewer

    老师自动指派；Action 随机抽同学 @ 提醒；也可在 PR 手动 @同学。改完记得点 Re-request review。
